Solution Outline

Make Roll-Up a two-stage pipeline

Deterministic extraction (facts/decisions/constraints) → “Canonical Roll-Up”

Optional LLM narration → “Editorial Roll-Up”

Lock a rule: LLM output can’t introduce new facts; it can only rephrase cited items

Add “truth anchors”: event IDs + hashes to detect drift

Keep the canonical artifact immutable; editorial can be regenerated anytime

Deep Dive
1) Split the artifact: Canonical vs Editorial

Canonical Roll-Up (truth artifact)

generated by script only

uses only: SQLite events + snapshot + constraints

includes:

event IDs

atomic facts (verbatim)

decisions (structured 4W+H)

constraints (verbatim)

thread status transitions

deterministic output, stable template

this is what you trust long-term

Editorial Roll-Up (readability artifact)

optional LLM pass

can improve phrasing, grouping, headings

MUST cite the canonical items it rephrases

can be deleted/regenerated without loss

This eliminates “summary-of-summary” because the editorial never becomes the input to the next roll-up. The canonical does.

2) Hard guardrails that make LLM safe(ish)

Rule A — No new facts

Every sentence in editorial must be traceable to one or more fact_id / decision_id / constraint references.

Rule B — Immutable constraints

Constraints are copied verbatim into both canonical + editorial.

Rule C — Drift detection

Canonical block contains a content_hash of:

selected event IDs

facts text

decisions fields

constraints text
If editorial deviates (adds claims without anchors), it fails validation or gets flagged.

3) What gets “rolled up” without summarizing

Instead of “summarize the day,” you promote stable items:

Promoted Facts: atomic facts that still matter after expiry

Decisions: the 4W+H record

Thread transitions: active → stalled → inactive → done

Warnings: failures, constraints, “don’t do this again”

Revisit conditions: triggers for reopening decisions
